Solution¶

i experienced a similar error recently. if you look at some similar situations that pivotal cloud foundry customers have encountered. talking with the engineers that work on cloud foundry and consul, they recommend:

stopping the monit job in each consul VM monit stop consul_agent removing all the consul data under rm -rf /var/vcap/store/consul_agent/* after all VMs cleared of consul agent data, starting monit in each one then it all came back for me.
